## 1. **Características da Plataforma**  
   

| Característica | Descrição |
| :---- | :---- |
| Descrição do Software | A plataforma é uma aplicação web voltada para o monitoramento do olhar dos usuários durante a visualização de propagandas, utilizando tecnologia de rastreamento ocular (eye tracking). Ela coleta dados de interação visual e gera relatórios analíticos para empresas, permitindo avaliar a efetividade das campanhas publicitárias. |
| Descrição do Hardware | O sistema é executado em computadores com webcam ou dispositivos de captura ocular (eye tracker). Servidores remotos armazenam e processam os dados coletados, e a interface web apresenta os resultados em relatórios visuais. |
| LISTA DE Capacidades da Plataforma (com explicação) | - **Coleta de Dados Visuais:** captura os pontos de fixação e movimentos oculares. <br> - **Armazenamento de Dados:** registra as informações em banco de dados para posterior análise. <br> - **Geração de Relatórios:** apresenta mapas de calor e estatísticas visuais. <br> - **Interface Intuitiva:** design centrado no usuário, permitindo fácil interpretação dos resultados. |
| LISTA DE Restrições da Plataforma (com explicação) | - **Dependência de Equipamento:** é necessário webcam funcional ou sensor ocular. <br> - **Conexão com Internet:** indispensável para upload e processamento de dados. <br> - **Ambiente de Iluminação:** o desempenho depende da iluminação e da posição do usuário. <br> - **Privacidade de Dados:** deve seguir a LGPD no tratamento de dados pessoais e biométricos. |

## 3. **Metas de Usabilidade**

### 1. **Qualitativo**

- Garantir **facilidade de uso** para usuários sem conhecimento técnico.  
- Fornecer **feedback visual imediato** após cada ação.  
- Criar **relatórios intuitivos e de fácil leitura**.

---

### 2. **Quantitativo**  
 
| Metas | Porcentagem | Justificativa |
| ----- | :---- | :---- |
| Facilidade de aprendizado | 25% | O sistema deve ser compreendido rapidamente sem treinamento. |
| Eficiência na execução de tarefas | 30% | A navegação e geração de relatórios devem ocorrer em poucos cliques. |
| Satisfação do usuário | 20% | O usuário deve sentir conforto e confiança durante o uso. |
| Taxa de erros mínima | 15% | A interface deve prevenir erros e permitir recuperação simples. |
| Acessibilidade | 10% | O design deve permitir acesso para usuários com diferentes perfis. |
| **Total** | **100%** |  |
